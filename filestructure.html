2. Source Files

2.1. PRobot\App.xaml

WPF application manifest. Declares PRobot.App as the application class and MainWindow.xaml as the startup window. Contains an empty Application.Resources dictionary — no global styles or resources are defined at the XAML level.

2.2. PRobot\App.xaml.cs

Code-behind for App.xaml. Partial class App : Application.

Responsibilities:

  • Forces InvariantCulture on the main thread and as the default for all subsequently spawned threads, ensuring that double.Parse / double.ToString always use . as the decimal separator regardless of the machine’s regional settings.

  • Attaches two global exception handlers:

    • DispatcherUnhandledException — catches exceptions that propagate up through the WPF message loop.

    • AppDomain.CurrentDomain.UnhandledException — catches exceptions on any thread.

    • Both handlers display a MessageBox and, for the dispatcher handler, mark the exception as handled (preventing the process from crashing on recoverable errors).

2.3. PRobot\MainWindow.xaml

XAML definition of the top-level window.

Key declarations:

Element Role

Window attributes

WindowState="Maximized", Background="#252530", d:Height="600", d:Width="1000".

Window.Resources

Defines default Style for Button, TextBlock, and Slider — dark theme colours.

Grid

Two rows (header 36 pt, content ) × two columns (viewport , settings panel 500 pt).

Border (header)

Spans both columns; holds the title StackPanel with "PRobot" and "Robot IK/FK Solver".

Border (viewport)

Name="mContent" — an empty Border whose Child is set to the Nori/WPF host element in the code-behind.

ScrollViewer

Right column; vertical scroll, no horizontal scroll; Background="#2A2A38".

StackPanel Name="mSettings"

Inside the scroll viewer; populated at runtime by RobotScene.CreateUI.

2.4. PRobot\MainWindow.xaml.cs

Code-behind for MainWindow.xaml. Partial class MainWindow : Window.

Responsibilities:

  • Constructor calls Lib.Init() — initialises the Nori framework (registers assemblies, sets up streams, etc.).

  • Constructor calls InitializeComponent() — WPF standard: instantiates the visual tree from XAML.

  • Constructor calls WPFHost.Init(this, OnLuxReady) and assigns the returned UIElement to mContent.Child, embedding the OpenGL surface inside the WPF layout.

  • OnLuxReady() callback (fired once the OpenGL context is active):

    • Sets the trace overlay text colour to yellow.

    • Creates a SceneManipulator (Nori built-in: handles mouse-driven pan/zoom/rotate in 3D scenes).

    • Creates RobotScene and assigns it to Lux.UIScene.

    • Defers scene.CreateUI(mSettings.Children) via Dispatcher.BeginInvoke so the settings panel is populated after the WPF layout pass has sized the columns.

2.5. PRobot\RobotScene.cs

The core of the application. Contains three classes (in three #region blocks):

RobotScene : Scene3

All robot logic — mechanism loading, FK/IK computation, collision detection, script playback, and UI construction. See Objects and Methods for the full API.

TcpVN : VNode

Streaming visual node that draws the TCP coordinate frame (three coloured arrows) and an origin ring every frame.

InfoVN : VNode

Streaming visual node that renders four lines of text in the lower-left corner of the viewport showing live joint angles and TCP position.

2.6. PRobot\Globals.cs

Eleven global using directives applied across the entire assembly:

global using System;
global using System.IO;
global using System.Linq;
global using System.Collections.Generic;
global using System.Windows;
global using Nori;

This file eliminates repetitive using statements in every other .cs file.

2.8. robot_script.txt

Plain-text waypoint script used by the script-playback feature.

Format: one waypoint per line — six space-separated floating-point numbers.

X  Y  Z  Rx  Ry  Rz

  • X Y Z — position offsets from the robot home in mm. Home is at world position (1166, 0, 596).

  • Rx Ry Rz — Euler rotation angles in degrees applied in order X → Y → Z.

  • Lines starting with # or blank lines are ignored.

The sample file defines a three-phase sequence: raise the tool, traverse a 300 × 200 mm rectangle, tilt the tool progressively, and return home.

3. Compiled Output (Bin\)

3.1. PRobot.exe (159 KB)

The .NET apphost launcher. Locates the correct .NET runtime, loads PRobot.dll, and calls its entry point. Because OutputType is WinExe, no console window is created.

3.2. PRobot.dll (31 KB)

The compiled managed assembly. Contains all types from the source files above. Small size reflects that the application is thin — all heavy lifting is in the Nori DLLs.

3.3. PRobot.pdb (20 KB)

Portable debug symbols. Maps IL offsets back to C# source lines. Enables stack traces with line numbers in the global exception handler.

3.4. PRobot.runtimeconfig.json

Tells the .NET host which frameworks the application requires at runtime:

"Microsoft.NETCore.App"    : "10.0.0"
"Microsoft.WindowsDesktop.App" : "10.0.0"

Also disables BinaryFormatter and Windows UI XAML projections (both deprecated).

3.5. PRobot.deps.json

Full dependency graph used by the runtime to locate assemblies. Records:

  • Direct dependencies: Nori.Core, Nori.Host.WPF, Nori.Lux.

  • Transitive NuGet packages: JetBrains.Annotations 2025.2.4, System.Reactive 6.0.1.

  • Assembly versions and file versions for all managed libraries.

4.1. Key Nori Source Files

4.1.1. N:\Core\Base\Lib.cs

Central framework class. Lib.Init() must be called before any other Nori API. Provides:

  • Tessellation delegate (Lib.Tessellate) used by mesh extrusion.

  • Trace delegate (Lib.Tracer) for debug output.

  • Resource loader (Lib.ReadBytes, Lib.ReadText, Lib.OpenRead) with nori: virtual scheme.

  • Mathematical constants and helpers.

4.1.2. N:\Core\Sim\Mechanism.cs

Defines the Mechanism class — a node in a kinematic tree. Mechanism.Load(path) parses a .curl file and returns the root node. Each node may have a joint type, limits, visual mesh, and collision mesh. The tree is traversed with EnumTree().

4.1.3. N:\Core\Sim\RBRSolver.cs

Closed-form IK for 6-DOF roll-bend-roll robots. Takes DH-style link parameters and per-joint limits in the constructor. ComputeStances fills an array of eight Soln records.

4.1.4. N:\Core\Sim\OBBTree.cs and OBBCollider.cs

Builds and tests OBB hierarchies. OBBTree.From(mesh) builds a tree using recursive median-cut partitioning. OBBCollider.Check runs SAT-based narrow-phase tests. Both use object pooling (IBorrowable<T>) to avoid per-frame allocation.

4.1.5. N:\Lux\Scene\Scene.cs

Scene3 (base class of RobotScene) provides a 3D turntable camera with Bound3 for auto-zoom, BgrdColor, and overridable hooks Detached() and Picked(object).

4.1.6. N:\Lux\VNode.cs

Base class for every visual object in the scene graph. Subclasses override Draw() and/or SetAttributes(). Streaming = true opts out of geometry caching so Draw() runs every frame.

4.1.7. N:\Wad\FanucX\mechanism.curl

Binary/text CURL file describing the FANUC X-series arm: joint names, rotation axes, limits, mesh file paths, and socket positions. Parsed by Mechanism.Load.
