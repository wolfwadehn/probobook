This document describes every class, struct, enum, property, and method that is defined in the PRobot source code and every type from the Nori framework that is directly used by PRobot. Types are listed in dependency order (leaf types first).

1. PRobot Application Layer

1.1. App (App.xaml.cs)

Partial class extending System.Windows.Application. Instantiated automatically by WPF when PRobot.exe starts.

1.1.1. Constructor — App()

public App()

  1. Sets Thread.CurrentThread.CurrentCulture and CultureInfo.DefaultThreadCurrentCulture to InvariantCulture. Ensures all double.Parse calls use . as the decimal separator, regardless of regional settings.

  2. Registers OnDomainUnhandledException on AppDomain.CurrentDomain.UnhandledException.

  3. Registers OnUnhandledException on this.DispatcherUnhandledException.

1.1.2. OnUnhandledException (static, private)

static void OnUnhandledException(object sender, DispatcherUnhandledExceptionEventArgs e)

Called when an unhandled exception propagates through the WPF dispatcher. Shows a MessageBox with the exception details. Sets e.Handled = true so the app keeps running.

1.1.3. OnDomainUnhandledException (static, private)

static void OnDomainUnhandledException(object sender, UnhandledExceptionEventArgs e)

Called for exceptions on non-dispatcher threads. Shows a MessageBox. Does not suppress further propagation (the CLR may still terminate the process).

1.2. MainWindow (MainWindow.xaml.cs)

Partial class extending System.Windows.Window. The sole top-level window.

1.2.1. Constructor — MainWindow()

public MainWindow()

  1. Lib.Init() — initialises the Nori framework.

  2. InitializeComponent() — standard WPF: builds the visual tree from MainWindow.xaml.

  3. WPFHost.Init(this, OnLuxReady) — creates the OpenGL surface and embeds it as mContent.Child.

1.2.2. OnLuxReady (private)

void OnLuxReady()

Callback invoked by Nori once the OpenGL context is active and the first frame can be rendered.

  1. Sets TraceVN.TextColor = Color4.Yellow.

  2. new SceneManipulator() — installs the built-in mouse handler for 3D pan/zoom/rotate.

  3. new RobotScene() — creates and initialises the complete robot scene.

  4. Lux.UIScene = scene — makes the scene the active rendered scene.

  5. Dispatcher.BeginInvoke) ⇒ scene.CreateUI(mSettings.Children — defers UI population until after the WPF layout pass, ensuring mSettings has its final size.

1.3. RobotScene (RobotScene.cs)

Extends Scene3 (from Nori.Lux). The application’s primary scene. Owns all robot logic.

1.3.1. Constructor — RobotScene()

RobotScene()

Full sequence described in Part Loading. Summary:

  1. Load mechanism from N:/Wad/FanucX/mechanism.curl.

  2. Locate Tip node and joints [S, L, U, R, B, T].

  3. Read JMin/JMax from each joint into mMin[] / mMax[].

  4. Construct RBRSolver with link lengths and joint limits.

  5. Build a 200 × 200 × 200 mm box obstacle (mesh + OBB tree + visual node).

  6. Build OBB trees for every robot link with a collision mesh.

  7. Set up gripper transform node.

  8. Set home pose and compute first IK solution.

  9. Create 500 ms DispatcherTimer for script playback.

  10. Create InfoVN and TcpVN overlay nodes.

  11. Assign Root scene graph.

1.3.2. CreateUI(UIElementCollection ui) (public)

public void CreateUI(UIElementCollection ui)

Programmatically builds the settings panel. Called once after the WPF layout pass.

  • Clears ui first.

  • Forward Kinematics section: iterates mMech.EnumTree(), creates one labelled Slider per joint. Slider callback sets m.JValue and calls OnFK().

  • Inverse Kinematics section: creates sliders for X, Y, Z, Rx, Ry, Rz. Special case: Rx slider starts at −90 (tool pointing up). Each callback updates the corresponding field and calls ComputeIK().

  • Stances section: adds mStances (a ListBox). SelectionChanged updates mSelStance and calls ComputeIK().

  • Obstacle section: sliders for BX, BY, BZ. Callbacks call UpdateBox().

  • Script section: TextBox for path, row of Button (Load, Play/Stop).

Local helper: AddSection(string text)

Creates a bold, light-steel-blue TextBlock as a visual separator.

Local helper: AddSlider(string label, double min, double max, double value, Action<double> setter)

Creates a horizontal StackPanel with a right-aligned 22 pt TextBlock label and a Slider. The slider fires setter(newValue) when changed, guarded by !mSyncingUI. Stores the Slider in mSliders[label].

1.3.3. Detached() (public override)

public override void Detached()

Stops the playback timer when the scene is removed from the renderer.

1.3.4. Picked(object obj) (public override)

public override void Picked(object obj)

Called by the Lux pick system when the user clicks in the viewport. If the clicked object is mBoxMesh, calls SnapToFace(Lux.PickPos).

1.3.5. SnapToFace(Point3 hit) (private)

void SnapToFace(Point3 hit)

Determines which face of the box was clicked by comparing the magnitude of the displacement from the box centre along each axis.

  • Identifies dominant axis → sets (mRx, mRy, mRz) to the corresponding face-normal orientation.

  • Sets (mX, mY, mZ) to the click point offset from home.

  • Calls ComputeIK() then SyncIKSliders().

1.3.6. ComputeIK() (private)

void ComputeIK()

  1. Sets mComputingIK = true to prevent re-entrant stance selection.

  2. Builds a CoordSystem from the Euler angles (mRx, mRy, mRz) applied in X → Y → Z order.

  3. Computes the TCP point as mHome.Org + Vector3(mX, mY, mZ), then offsets 50 mm along the approach axis (cs.VecZ * 50) to account for tool length.

  4. Clears and repopulates mStances.Items (8 entries, one per IK branch).

  5. Calls mSolver.ComputeStances(mCS.Org, mCS.VecZ, mCS.VecX).

  6. For the selected stance, sets mJoints[i].JValue for all six joints.

  7. Updates mGripper.Xfm = mTip.Xfm.

  8. Calls CheckCollisions().

  9. Sets mComputingIK = false.

1.3.7. OnFK() (private)

void OnFK()

Called after any FK slider change. Updates the gripper transform and checks collisions.

1.3.8. SyncIKSliders() (private)

void SyncIKSliders()

Sets the IK sliders to the current mX/mY/mZ/mRx/mRy/mRz values while suppressing the ValueChanged callbacks via mSyncingUI. Clamps values to the slider range.

1.3.9. BoxWorldXfm (private property)

Matrix3 BoxWorldXfm => Matrix3.Translation(mBX, mBY, mBZ)

Computes a pure translation matrix positioning the box in world space.

1.3.10. UpdateBox() (private)

void UpdateBox()

Applies the new box world transform to mBoxXfm.Xfm and calls CheckCollisions().

1.3.11. CheckCollisions() (private)

void CheckCollisions()

  1. Computes world-space box OBB: mBoxOBB.With(BoxWorldXfm).

  2. Borrows an OBBCollider from the pool.

  3. For each (m, linkOBB) pair in mLinkOBBs:

    1. Computes world-space link OBB: linkOBB.With(m.Xfm).

    2. Calls bc.Check(worldLinkOBB, worldBoxOBB).

    3. Sets m.IsColliding = hit.

    4. Tracks anyHit.

  4. Sets mBoxVN.Color = anyHit ? Color4.Red : Color4.Blue.

1.3.12. LoadScript(string path) (private)

void LoadScript(string path)

Reads the waypoint file line by line. Skips blank lines and # comments. Splits each remaining line on whitespace, parses six doubles using InvariantCulture, and appends a named-tuple (X, Y, Z, Rx, Ry, Rz) to mScript.

1.3.13. TogglePlay() (private)

void TogglePlay()

If the timer is running, stops it and resets the button label to "Play". If not running, resets mScriptIdx to 0, starts the timer, and labels the button "Stop".

1.3.14. TickScript() (private)

void TickScript()

Called by mPlayTimer every 500 ms. Reads the next waypoint from mScript, assigns it to the six IK fields, and calls ComputeIK(). Stops the timer when the script is exhausted.

1.4. TcpVN (RobotScene.cs)

Extends VNode (Nori.Lux). Streaming = true — Draw() executes every frame.

1.4.1. Constructor — TcpVN(RobotScene scene)

Stores the scene reference. Sets Streaming = true.

1.4.2. SetAttributes() (public override)

public override void SetAttributes()

Sets Lux.ZLevel = 70 — renders on top of most scene geometry.

1.4.3. Draw() (public override)

public override void Draw()

  1. Obtains the current TCP frame: fcs = CoordSystem.World * scene.Tip.Xfm.

  2. Computes the TCP tip position: tcp = fcs.Org - fcs.VecZ * 50 (back-projects 50 mm tool offset).

  3. Calls DrawArrow(tcp, fcs.VecX, Red), DrawArrow(tcp, fcs.VecY, Green), DrawArrow(tcp, fcs.VecZ, Blue).

  4. Draws a 24-segment white ring of radius 10 mm in the TCP XY plane using Lux.Lines(ring.AsSpan()).

1.4.4. DrawArrow(Point3 from, Vector3 dir, Color4 color) (private)

void DrawArrow(Point3 from, Vector3 dir, Color4 color)

Draws a simple 3D arrow with a line body and two "fins" forming the head.

  • Shaft length L = 90 mm. Head length HL = 15 mm. Head half-width HW = 7 mm.

  • Computes a perpendicular side vector (avoids the degenerate case when dir ≈ XAxis).

  • Emits six Vec3F vertices: shaft start → tip, two head fins.

1.5. InfoVN (RobotScene.cs)

Extends VNode (Nori.Lux). Streaming = true.

1.5.1. Constructor — InfoVN(RobotScene scene)

Stores scene reference, sets Streaming = true. Constructs a TypeFace from the embedded RobotoMono-Regular.ttf at 14 pt.

1.5.2. SetAttributes() (public override)

public override void SetAttributes()

Sets Lux.Color = White, assigns the TypeFace, sets Lux.ZLevel = 90 (renders on top of everything including TcpVN).

1.5.3. Draw() (public override)

public override void Draw()

  1. Guards against null scene.

  2. Reads InfoData from the scene.

  3. Computes the starting Y position: 4 lines from the bottom of the viewport: (int)sc.Rect.Height - lh * 4 - 8.

  4. Renders four text lines via Lux.Text(s, new Vec2S(px, py)), advancing py by lh each time:

    Line 1: X=<absX>  Y=<absY>  Z=<absZ>  mm
Line 2: Rx=<rx>°  Ry=<ry>°  Rz=<rz>°
Line 3: S=<s>°  L=<l>°  U=<u>°
Line 4: R=<r>°  B=<b>°  T=<t>°

2.2. RBRSolver (N:\Core\Sim\RBRSolver.cs)

Closed-form inverse kinematics solver for 6-DOF roll-bend-roll wrist robots. Computes all eight algebraic solution branches in one call.

RBRSolver(double a12, double a23, double a34,
